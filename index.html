Making songs available via Coria

This is a demo of two ways to make practice material available for your choir via Coria.

For both of these two ways you need to have a website of some kind. You also need to have the song or songs that you want to share in format that is supported by coria.

The website can be very simple. This example uses github pages, which is free. But if your choir has its own website you should be able to use that.

The formats that are supported by coria are musicXML and Noteworthy Composer format. A popular free option to create musicXML is Musescore. Many musicXML files for choral music can be downloaded from the site cpdl.org. Noteworthy composer (NWC) files can be created using the tool Noteworthy Composer.

Method 1: publishing songs as web-pages

For this example the music file has been converted to a .html file. This .html file needs to be placed onto a webserver, and you can simply link to it.

You can convert a music file (MusicXML or NWC) using the coria website (https://coria.nl).

  1. Select the option "Use your own file" ("Gebruik eigen bestand", in Dutch. If you want an English version then change the language that Coria uses to English, this can be done on the homepage);
  2. select the file that you want to convert;
  3. select one of the voices (it doesn't matter which one), and press "play";
  4. ensure that the result is what you expected;
  5. select the option "download file for distribution (html format);
  6. put the html file on your webserver and communicate a link to it. You can use a link from your website, but it could also be a link in an email.

In this particular example the downloaded file was placed on this same webserver, in the "music" folder. So the link below is a relative link. In the html source file for this page it looks like this: <a href="music/Matona_mia_cara.html">Matona mia cara</a>

Example: Matona mia cara

This method has an advantage over method 2 below: If your website supports it, you can put the html page in section of the website that requires the user to log in. That way you can manage who has access to the page. With option 2 that is not possible, because coria must be able to access the music file, so the url pointing to the music file must be publicly available.

Method 2: linking to song files

The alternative to creating the self contained html file, is to publish the source music file, aswell as a link to coria that refers to the source file.

Such a link could look like this: <a href="https://coria.nl/play_from_url?url=https://willemdj.github.io/coria/music/Lassus_Matona_mia_cara.xml">Matona mia cara</a>. The first bit of the link, https://coria.nl/play_from_url, refers to a special coria page. This page will take everything that comes after the & character and interpret it as a link to a music file. It uses the link to fetch the file, translates it so that coria can use it, and enables the user to play the piece.

Example: Matona mia cara

In most cases, method 1 above is preferable over this method. With method 1 you can put the page in a protected area of your website, and you don't have to make the source file (MusicXML or NWC file) publicly available. However, there may be cases where this second option is less work, for example if publication of the music files to everybody who might be interested is the main goal of your website.
